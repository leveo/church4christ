**Architecture:** New `children` module following every existing convention: D1+Postgres mirrored migration, `src/lib/checkinDb.ts` against the `AppDb` seam, server-rendered Astro pages with inline form-POST handling, no client framework, no new dependencies. **Tech Stack:** Astro 7 SSR on Cloudflare Workers, D1/Postgres via AppDb adapter, Tailwind v4 + design tokens, Vitest (workers pool + e2e).

```ts
import type { AppDb } from './appDb';

export interface CheckinEvent {
  id: number; name: string; weekday: number | null; active: number;
}
export interface KioskHouseholdHit {
  id: number; name: string; phone: string | null;
  adults: string[]; children: string[];
}

export function listEventsAdmin(db: AppDb): Promise<CheckinEvent[]>;                    // all, newest first
export function listActiveEvents(db: AppDb, weekday: number): Promise<CheckinEvent[]>;  // active AND (weekday IS NULL OR weekday = ?)
export function saveEvent(db: AppDb, input: { id?: number; name: string; weekday: number | null }): Promise<number>; // returns id
export function toggleEventActive(db: AppDb, id: number, active: boolean): Promise<void>;
export function searchHouseholds(db: AppDb, q: string): Promise<KioskHouseholdHit[]>;   // LIMIT 10
```

**Behavior contract for `searchHouseholds`:**
- `digits = q.replace(/\D/g, '')`; if `digits.length >= 4` → phone mode, else name mode. Empty/whitespace `q` → `[]`.
- Name mode: live households (`h.deleted_at IS NULL`) having a `household_members` row with `role='child'` and `LOWER(display_name) LIKE '%'||LOWER(q)||'%' ESCAPE '\'` (escape `%_\` in JS first — copy the escaping from `src/lib/adminDb.ts:88-93`).
- Phone mode: live households where digit-stripped `households.phone` contains `digits`, OR an adult member's `people.phone` does. Digit-strip in SQL with nested `REPLACE(col,'-','')` chains removing `-`, `' '`, `(`, `)`, `+`, `.`.
- Each hit carries all its adults' and children's `display_name`s (second query over the matched household ids, grouped in JS; keep it two queries, not N+1).
- Only households that HAVE at least one child appear in results (both modes).

**Files:**

```ts
export interface KioskChildStatus {
  memberId: number; name: string;
  checkins: { checkinId: number; eventId: number; eventName: string; securityCode: string; checkedOutAt: string | null }[];
}
export interface RosterRow {
  checkinId: number; childName: string; householdName: string; eventName: string;
  securityCode: string; checkedInAt: string; checkedOutAt: string | null;
}
export interface WeeklyStats {
  weeks: { weekStart: string; total: number }[];               // oldest → newest, exactly weeksBack entries, zero-filled
  byEvent: { eventId: number; name: string; counts: number[] }[]; // counts aligned to last 4 entries of weeks
  thisWeek: number; fourWeekAvg: number; distinctChildrenThisMonth: number; activeEvents: number;
}

export function getHouseholdForKiosk(db: AppDb, householdId: number, date: string):
  Promise<{ id: number; name: string; children: KioskChildStatus[] } | null>;  // null if household deleted/missing
export function checkInChildren(db: AppDb, input: { eventId: number; householdId: number; memberIds: number[]; date: string }):
  Promise<{ code: string; checkedIn: string[] }>;   // throws Error('no_children') if no valid member ids
export function checkOutChild(db: AppDb, input: { checkinId: number; code: string }): Promise<boolean>; // false = wrong code / already out / not found
export function staffCheckOut(db: AppDb, checkinId: number): Promise<void>;
export function todayRoster(db: AppDb, date: string): Promise<RosterRow[]>;
export function weeklyStats(db: AppDb, opts: { today: string; weeksBack?: number }): Promise<WeeklyStats>; // default weeksBack 12
export function generateSecurityCode(rand?: () => number): string; // exported for tests
```

**Behavior contract:**
- `generateSecurityCode`: 4 chars from `'ABCDEFGHJKMNPQRSTUVWXYZ23456789'` (no 0/O/1/I/L).
- `checkInChildren`: (1) validate memberIds → `SELECT id, display_name FROM household_members WHERE household_id = ? AND role = 'child' AND id IN (…)` (build `?` list); (2) reuse existing code: `SELECT security_code FROM checkins WHERE event_id=? AND household_id=? AND checkin_date=? LIMIT 1`, else `generateSecurityCode()`; (3) insert one row per child sequentially, swallowing `isUniqueViolation` (already checked in = success); (4) return code + ALL validated child names.
- `checkOutChild`: `UPDATE checkins SET checked_out_at = datetime('now') WHERE id = ? AND checked_out_at IS NULL AND UPPER(security_code) = UPPER(?)`; return `meta.changes > 0` (PgAdapter sets `meta.changes`).
- `weeklyStats`: one `GROUP BY checkin_date, event_id` query over `checkin_date >= <oldest weekStart>`, then bucket in JS. Week starts **Sunday**: `weekStartOf(dateStr)` = shift back by `new Date(dateStr+'T00:00:00Z').getUTCDay()` days using `addDays` from `src/lib/dates.ts`. `distinctChildrenThisMonth`: `COUNT(DISTINCT household_member_id)` where `checkin_date LIKE 'YYYY-MM-%'` of `today`. `fourWeekAvg`: mean of the last 4 completed values incl. this week, rounded.
