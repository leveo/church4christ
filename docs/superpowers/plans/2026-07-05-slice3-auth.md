## Global Constraints

- Tokens: store sha256 hex hash ONLY; login TTL 15 min; respond TTL 14 days; rate limit
  3 login tokens / 15 min / person; anti-enumeration (signin always renders "sent").
- Two-step consume: GET `/auth/[token]` peeks (no mutation), POST consumes atomically
  (`UPDATE tokens SET used_at = datetime('now') WHERE token_hash = ? AND used_at IS NULL
  AND expires_at > datetime('now') RETURNING person_id` style).
- Session cookie `c4c_session`: HttpOnly, SameSite=Lax, Path=/, Secure when
  `import.meta.env.PROD`, Max-Age 30d; JWT claims `sub` (person id), `email`, `ep`.
- Middleware order: (1) locale handling (exists, slice 1), (2) attach
  `locals.user: SessionUser|null` — reload person row every request; reject when
  `active=0`, `deleted_at`, or `ep` mismatch, (3) route policy gate, (4) CSRF origin
  check for non-GET (403 on cross-origin; allow same-origin and `Sec-Fetch-Site:
  same-origin`), (5) security headers + `no-store` when user attached.
- `SessionUser` (replaces slice-1 stub in `src/lib/types.ts`):
  `{ id, email, displayName, role: 'member'|'editor'|'admin', isAdmin, isEditor,
  memberTeamIds: number[], leaderTeamIds: number[], lang: 'en'|'zh'|null }`.
- Route policy (pure, `src/lib/routePolicy.ts`): classify a pathname (locale prefix
  stripped) into `public | authed | team | console | adminOnly`:
  - public: `/`, `/visit`, `/about*`, `/articles*`, `/fellowships*`, `/ministries*`,
    `/events`, `/sermons*`, `/bulletin*`, `/prayer*`, `/give`, `/privacy`, `/serve`,
    `/serve/gifts`, `/serve/apply`, `/serve/testimonies`, `/signin`, `/auth/*`,
    `/respond/*`, `/cal/*`, `/media/*`, `/api/prayer-request`, `/healthz`, `/404`
  - authed: `/my*`, `/profile`, `/settings/*`
  - team (member∪leader∪admin): `/serve/plans*`, `/serve/matrix*`, `/serve/teams*`,
    `/profile/[id]` (leader∪admin only — policy returns `team` and page enforces finer)
  - console (editor∪admin∪leader): `/admin` + CMS content routes `/admin/{bulletins,
    sermons,prayer-sheets,announcements,events,prayer-wall,revisions}*` (editor∪admin),
    volunteer console `/admin/{ministries,availability,applications}*` (leader∪admin)
  - adminOnly: `/admin/{people,service-types,settings,reports,teams}*`
  Unknown-path fallback is namespace-scoped: unknown under `/admin` → `adminOnly`;
  under `/my`,`/profile`,`/settings` → `authed`; under `/serve` → `team`; all other
  unknown paths → `public` (natural 404 — a public site must not bounce typo URLs to
  signin). Unauthenticated access to non-public → 303 to
  `/{locale}/signin?next=…` for GET, 403 for others.
- Email (`src/lib/email.ts`): `sendEmail(env, db, {to, toName, kind, subject, html, text,
  detail}) -> Promise<boolean>`; uses `env.EMAIL.send` (EmailMessage from
  `cloudflare:email`, MIME via raw string building — port dcfc-serve exactly); when
  `env.EMAIL_DEV_LOG === '1'` log to console + email_log status `devlog` and skip send;
  always best-effort (never throw to caller); log sends to email_log.
- Dev bypass: in middleware, when `import.meta.env.DEV` and `AUTH_DEV_BYPASS_EMAIL` set
  and no session cookie → auto-attach that person's session (no cookie mutation).
- No PII/DCFC values. Conventional commits ending
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
